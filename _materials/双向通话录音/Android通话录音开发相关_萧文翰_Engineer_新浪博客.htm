Android通话录音开发相关

(2014-01-15 08:55:38)
转载▼
分类： Android开发相关
本文主要讲述的是Android平台下如何实现自动通话录音功能。
需要注意的是，这个功能由于各机器ROM的限制，有可能造成无法使用的情况，这个还是有待改善的。

先说一下原理：
1、在电话被接通、被挂断时，接收系统的广播；
2、当电话被接通后，开始录音；
3、当电话被挂断时，结束录音并释放相关资源；
4、为了实现自动录音，要求程序开机自启。

为了正常接收系统广播并实现开机启动，需要为程序注册广播接收器。

该类继承自BroadcastReceiver，再重写的onReceive方法中，根据广播种类进行相应操作。具体如下：
if (intent.getAction().equals(action)) {
// 开机启动
Intent inte = new Intent();
// 跳到通话录音服务
inte.setAction(".service.CallService");
inte.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
// 添加一个activity的堆空间，否则程序报错
context.startService(inte);
} else if (intent.getAction().equals(Intent.ACTION_NEW_OUTGOING_CALL)) {
// 判断是不是向外打电话
Intent inte = new Intent();
// 跳到通话录音服务
inte.setAction(".service.CallService");
inte.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
// 添加一个activity的堆空间，否则程序报错

} else {
TelephonyManager tm = (TelephonyManager) context.getSystemService(Context.TELEPHONY_SERVICE);
if (tm.getCallState() == TelephonyManager.CALL_STATE_RINGING) {
// 判断是不是有来电
Intent inte = new Intent();
// 跳到通话录音服务
inte.setAction(".service.CallService");
inte.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
// 添加一个activity的堆空间，否则程序报错
context.startService(inte);
}
}
另外，为了保证程序能够正常监听来电状态和实现录音，还需在AndroidManifest.xml文件中做权限及相应服务注册：

android:name="com.xwh.callrecorder.service.CallService"
android:label="录音服务"
android:process=".message" >
       
这样，就实现了开机自启并监听来电状态。
